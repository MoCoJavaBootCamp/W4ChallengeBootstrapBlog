Scaling Your Application via Reverse Proxy

May 14, 2020 by Andrew Doolittle

Scaling your application takes on a variety of design decisions. You may want faster speeds, more throughput, or better security. One way you can optimize your system is by implementing a reverse proxy server.

Reverse proxies balance loads by intercepting client requests, caching files, and distributing server calls

HTTP Request Basics

When a user visits a website, a number of things happen. First, they enter a URL like CodingWithAndrew.com. Behind the scenes, the user's browser will find an IP address associated with that URL and make an HTTP request for the IP address' HTML page. After receiving the HTML page, the browser will read the code inside the HTML page.

Most likely, there are additional links and scripts needed, like retrieving the Bootstrap CDN, a CSS file, or running the JavaScript. That JavaScript code will make another request to the main server. Then, depending on what is needed in that request, a photo, a redirect to a product page, or a confirmation of a payment, the main server will make a request to another server dedicated to specific tasks.

A photo request leads to the database server hosting photos; a product page request leads to the database server hosting information on products; a payment confirmation request leads to another server handling payment process logic.

Need For Scaling

Each request may jump around two or more servers. While it doesn't seem like much, Internet traffic for a global shopping website like Amazon with millions of users and millions of products means merely visiting the Amazon home page already contributes to billions of requests and that slows things down.

So rather than a request jumping around multiple servers, a well configured reverse proxy will determine which requests are the most popular, save information from the databases, intercept those calls, and directly return the requests.
