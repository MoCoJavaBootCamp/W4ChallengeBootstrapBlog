Classes in Java

May 4, 2020 by Andrew Doolittle

Building off the idea that nearly everything in Java is an object, we can begin to understand classes.

Just like in real life where many things share the same material or genetic makeup, so too can objects in Java. Where a golden retriever is a dog, which is a type of canine, coming from mammals in the larger sphere of animals. Each of these has a parent to child relationship as they are not completely unrelated.

Let's take a look at the following code examples to understand how classes are created in Java.

public class Animal {
   public Animal() {
      System.out.println("A new animal has been created!");
   }
   public String sleep() {
      return "An animal sleeps...";
   }
   public String eat() {
      return "An animal eats...";
   }
}

public class Bird extends Animal {
    public Bird() {
        //super();
        System.out.println("A bird is hatched...");
    }

    @Override
    public String sleep() {
        return "A bird sleeps soundly...";
    }

    @Override
    public String eat() {
        return "A bird eats...";
    }


    public String fly() {
        //Birds can fly but other animals can't so this method
        //is unique to the Bird class
        return "A bird flies...";
    }
}

public class AnimalApp {

    public static void main(String[] args) {

        Animal a = new Animal(); // "A new animal has been created!"
        print(a.eat()); // "An animal eats..."
        print(a.sleep()); // "An animal sleeps..."

        Bird b = new Bird();
        print(b.eat()); // "A bird eats..."
        print(b.sleep()); // "A bird sleeps soundly..."
        print(b.fly()); // "A bird flies..."

    }
}
