(Nearly) Everything in Java is an Object

May 4, 2020 by Andrew Doolittle

According to Wikipedia:

An object can be a variable, a data structure, a function, or a method, and as such, is a value in memory referenced by an identifier.

By assigning objects as a value in memory, developers are able to design modular and reusable software systems. Not reusing code is inefficient as the system would need to create separate reservations in memory for similar or identical things.

Imagine we're drawing a simple picture of a picnic at the beach using crayons. To color the sun, we use yellow. To color the sand, we use light brown. To color the water, we use blue.

However, our box has 5 blue crayons of exactly the same shade. Which one do we use? To a child, they're all the same, but to a computer, they are 5 separate unrelated objects.
